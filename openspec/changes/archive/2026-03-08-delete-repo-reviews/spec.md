> **Change**: `delete-repo-reviews`

## Summary

This spec defines the requirements for deleting all reviews for a specific repository, with an optional flag to also clear memory observations. It covers three layers: database query, server API endpoint, and dashboard UI.

## Cross-Cutting Concerns

### Authorization Model

All delete operations MUST follow the existing authorization pattern:
1. Look up repository by `fullName`
2. Verify `repo.installationId` is in the authenticated user's `installationIds`
3. Return 404 if repo not found, 403 if user lacks access

### API Envelope

- Success: `{ data: { deletedReviews: number, clearedMemory: number | null } }`
- Error: `{ error: 'ERROR_CODE', message: string, errorId?: string }`
- All 500 responses MUST include `errorId` generated by `generateErrorId()`

### Confirmation Tier

The dashboard delete action uses **Tier 2** confirmation (user must type the exact repository full name). This is consistent with the existing "Clear Memory" per-repo action.

### Cache Invalidation

On successful deletion:
- `['reviews']` — always invalidated
- `['stats', repoFullName]` — always invalidated (stats are computed from reviews)
- `['memory']` — only invalidated when `includeMemory` was true
