> **Change**: `delete-repo-reviews`
> **Base spec**: `openspec/specs/server/spec.md`

## ADDED Requirements

### Requirement: Delete Reviews By Repository

The system MUST expose a `DELETE /api/reviews/:repoFullName` endpoint that deletes all reviews for a given repository, with an optional flag to also clear memory observations in the same request.

The `:repoFullName` parameter MUST be URL-encoded (e.g., `acme%2Fwidgets` for `acme/widgets`). The server MUST decode it before use.

#### Scenario: Delete all reviews for a repository (happy path)

- GIVEN an authenticated user who owns the installation for repository "acme/widgets"
- AND the repository has 15 reviews in the `reviews` table
- WHEN `DELETE /api/reviews/acme%2Fwidgets` is called
- THEN the response MUST be HTTP 200 with body `{ data: { deletedReviews: 15, clearedMemory: null } }`
- AND all 15 reviews for that repository MUST be deleted from the database
- AND reviews for other repositories MUST NOT be affected

#### Scenario: Delete reviews with includeMemory flag

- GIVEN an authenticated user who owns the installation for repository "acme/widgets"
- AND the repository has 10 reviews and 25 memory observations
- WHEN `DELETE /api/reviews/acme%2Fwidgets?includeMemory=true` is called
- THEN the response MUST be HTTP 200 with body `{ data: { deletedReviews: 10, clearedMemory: 25 } }`
- AND all reviews for that repository MUST be deleted
- AND all memory observations for project "acme/widgets" MUST be cleared (via the existing `clearMemoryObservationsByProject` function)

#### Scenario: Delete reviews without includeMemory flag (default behavior)

- GIVEN an authenticated user who owns the installation for repository "acme/widgets"
- AND the repository has 5 reviews and 30 memory observations
- WHEN `DELETE /api/reviews/acme%2Fwidgets` is called (no query parameter)
- THEN the response MUST be HTTP 200 with body `{ data: { deletedReviews: 5, clearedMemory: null } }`
- AND memory observations MUST NOT be affected

#### Scenario: Delete reviews for a repository with zero reviews

- GIVEN an authenticated user who owns the installation for repository "acme/widgets"
- AND the repository has 0 reviews
- WHEN `DELETE /api/reviews/acme%2Fwidgets` is called
- THEN the response MUST be HTTP 200 with body `{ data: { deletedReviews: 0, clearedMemory: null } }`

#### Scenario: Repository not found returns 404

- GIVEN an authenticated user
- WHEN `DELETE /api/reviews/nonexistent%2Frepo` is called
- AND no repository with `fullName = "nonexistent/repo"` exists in the database
- THEN the response MUST be HTTP 404 with body `{ error: "NOT_FOUND", message: "Repository not found" }`

#### Scenario: User lacks access to repository installation returns 403

- GIVEN an authenticated user whose `installationIds` do NOT include the installation that owns repository "other-org/private-repo"
- WHEN `DELETE /api/reviews/other-org%2Fprivate-repo` is called
- THEN the response MUST be HTTP 403 with body `{ error: "FORBIDDEN", message: "Forbidden" }`
- AND no reviews MUST be deleted

#### Scenario: Unauthenticated request returns 401

- GIVEN a request with no Bearer token or an invalid/expired token
- WHEN `DELETE /api/reviews/acme%2Fwidgets` is called
- THEN the response MUST be HTTP 401
- AND no reviews MUST be deleted

#### Scenario: Internal server error returns 500 with errorId

- GIVEN the database is unreachable or the delete query throws an unexpected error
- WHEN `DELETE /api/reviews/acme%2Fwidgets` is called
- THEN the response MUST be HTTP 500 with body `{ error: "DELETE_FAILED", message: "Failed to delete reviews", errorId: "<uuid-prefix>" }`
- AND the `errorId` MUST be generated by `generateErrorId()`
- AND the error MUST be logged with the `errorId` for correlation

#### Scenario: Memory clear fails but review deletion succeeds (partial failure)

- GIVEN an authenticated user who owns the installation for repository "acme/widgets"
- AND the repository has 10 reviews
- AND `clearMemoryObservationsByProject` throws an error
- WHEN `DELETE /api/reviews/acme%2Fwidgets?includeMemory=true` is called
- THEN the response MUST be HTTP 500 with body `{ error: "DELETE_FAILED", message: "Failed to delete reviews", errorId: "<uuid-prefix>" }`
- AND the error MUST be logged

### Requirement: Database Query for Bulk Review Deletion

The system MUST provide a `deleteReviewsByRepoId` function in the database query layer (`packages/db/src/queries.ts`).

The function MUST accept a `Database` instance and a `repositoryId` (number) and MUST return the count of deleted rows (number).

The function MUST delete from the `reviews` table where `repositoryId` matches the given value.

#### Scenario: Delete reviews by repository ID

- GIVEN a repository with `id = 42` that has 20 reviews in the `reviews` table
- WHEN `deleteReviewsByRepoId(db, 42)` is called
- THEN all 20 reviews with `repositoryId = 42` MUST be deleted
- AND the function MUST return `20`
- AND reviews with other `repositoryId` values MUST NOT be affected

#### Scenario: Delete reviews for repository with no reviews

- GIVEN a repository with `id = 99` that has 0 reviews
- WHEN `deleteReviewsByRepoId(db, 99)` is called
- THEN the function MUST return `0`
- AND no database error MUST occur
